HTML

  • The head element contains information about the webpage.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users and is not typically seen by users.
  • The body element represents the visible content shown to the user.
  • Semantic HTML elements is important because it makes HTML easier to read by other developers and the browser and allows the webpage to be accessible to a wider variety of users.
  • The link element connects an external resourse to the HTML document.
  • The script element embeds client-side scripts in the HTML document.
  • The charset element sets our character encoding.
  • The Viewport element sets the default view for our webpage.
  • Section elements define a section in a document that contains thematic content.

CSS

  • CSS is a programming language used by devlopers to define how webpage content is presented to users.
  • Inline CSS apply to a specific HTML tag, using a style attribute wuth a CSS rule to style a particular page element only.
  • Internal style sheets contain CSS rules for the webpage in the head section of the HTML file that only apply to that page.
  • External CSS style sheets allows developers to keep all their CSS rules in a seperate file, making design changes easier.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The style element defines the base URL for a webpage.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -a: adds modifications in the current working branch to the staging area
  • git commit -m: commits changes. add message to describe changes.
  • git pull origin main: pulls command to receive a branches modifications to local environment
  • git push origin: pushes changes we have locally to our remote github branch
  • A pull request is used to merge a comparison branch into the base branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Control flow can be interrupted but conditional statements and loops and be used to control the order in which a code is executed or not.
  • An if statement will only execute a code if the statement meets a condition or is true.
  • Strings are a sequence of character, either as a literal constant or as some kind of variable.
  • Strings are always wrapped in quotation marks.
  • An array is a single variable that is used to hold a group of data.
  • Index order always goes in order from 0, 1, 2, 3, etc.
  • For loops contain 3 statements, the starting point of a loop, the condition, and what allows the array to itterate over each item.
